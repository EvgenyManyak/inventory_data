🗺 Roadmap: Inventory Analytics Pipeline (Hybrid Edition)
Этап 1: Инфраструктура в Docker
Стек: PostgreSQL + Python Container + Volumes.
Задача: Поднять базу и подготовить общие папки: /inbox (файлы из 1С), /processed (сведенные файлы) и /out (готовые PDF).
Результат: Все инструменты изолированы и готовы к работе.
Этап 2: Python Ingestion (Импорт из Excel в БД)
Стек: Pandas, SQLAlchemy.
Задача: Скрипт-загрузчик. Скачивание файлов из 1С вручную и перемещение в папку. Python считывает их и копирует в БД (таблицы raw_1c_data).
Результат: Все данные из "кривых" Excel-выгрузок лежат в структурированном SQL.
Этап 3: Аналитика и Сведение (SQL + Manual)
Стек: PostgreSQL, DBeaver или psql.
Задача: Подключение к БД через Python для выполнения SQL-запросов.
Логика: Сведение (учитывая нюансы компании). Python помогает выгрузить результат этого сведения обратно в Excel для финальной проверки.
Результат: Чистый, сведенный файл, подтвержденный SQL-расчетами.
Этап 4: Генератор PDF-отчетов (Python Reporting)
Стек: Matplotlib (диаграммы), FPDF2.
Задача: Python берет уже сведенный результат из БД, превращает его в PDF-отчет: мини-таблицы по ТТ + текст выводов + графики потерь.
Результат: Профессиональный PDF, готовый к отправке.
Этап 5: Автоматизация рассылки и Архивация
Стек: aiogram (Telegram Bot), shutil (архивация).
Задача: Доставка и порядок.
Бот отправляет PDF и сведенный Excel руководителю.
Python переносит исходники в архив и очищает рабочие папки.
Результат: Полный цикл закрыт.
